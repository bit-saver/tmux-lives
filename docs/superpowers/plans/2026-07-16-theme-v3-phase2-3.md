# Theme v3 Phases 2+3 (always-on cutover + theme picker) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the gradient map the always-on engine (v2 geometric engine deleted, old universals migrated, ShellFish tabs wear the `tabs` role) and replace the cap-picker with the layout-A theme picker (scheme catalog + live fake-bar preview).

**Architecture:** Phase 2 first: flip the default (`tmux_lives_theme` unset → `mono`, `off` becomes a stored legacy-look token), delete the v2 engine from both files, renumber the fragment argv, add the idempotent migration shim to the fisher post-update handler, and resolve the ShellFish tab colour from the live `@tmux_lives_tabs_color` option. Phase 3 then builds the new picker on the proven `__tcz_cap_picker` skeleton (raw-tty readkey loop, one config-loaded `fish -c` batch for palettes, signal-safe cleanup, frame-exactly-fills-popup) with a fake status-bar preview row rendered from the cursor row's palette. Spec: `docs/superpowers/specs/2026-07-16-theme-v3-phase2-3-design.md` (decisions resolved — do NOT re-litigate).

**Tech Stack:** fish 4.7.1, tmux 3.3a; the shipped Phase-1 engine (`__tmux_lives_theme_arc/_roles/_sample/_lrange/_palette`, `__tmux_lives_seed_hex` in `conf.d/tmux-lives-install.fish`), `__tcz_theme` UI-chrome palette + `__tcz_popup_readkey`/`__tcz_strip_sgr` in `functions/tmux-categorize.fish`. No new files, no new dependencies.

## Global Constraints

- **Always-on semantics (locked):** the effective theme is `__tmux_lives_key tmux_lives_theme mono` — unset = `mono`. The stored token `off` = the legacy look: `status-style` from `__tmux_lives_derive_status`, `@tmux_lives_bar_bg` = derived bg (`colour236` fallback), `@tmux_lives_cap_bg colour238` + `@tmux_lives_cap_fg` `#f5f5f5`, role options `default`/`''`. A stored scheme with no usable seed renders the same legacy branch.
- **Migration (locked):** idempotent, in `_tmux_lives_post_update`, BEFORE the fragment re-render. `cap_vividness` maps `subtle`→`soft` (`balanced`/`vivid` verbatim; anything else dropped); `cap_key`→`theme_key` verbatim; then erase `tmux_lives_cap`, `tmux_lives_cap_vividness`, `tmux_lives_cap_wheel`, `tmux_lives_cap_role`, `tmux_lives_cap_key`; if a cap scheme existed print exactly one line: `tmux-lives: cap scheme '<x>' has no v3 equivalent — theme is mono; tune with 'tmux-lives setup theme'`. Never touch `tmux_lives_theme*` values that already exist.
- **Removal is total:** after Phase 2 no reference to `__tmux_lives_palette`, `__tmux_lives_target_hue`, `__tmux_lives_interp7`, `__tmux_lives_rgb_to_ryb_hue`, `__tmux_lives_ryb_to_rgb_hue`, `__tmux_lives_hsl_hue`, `__tmux_lives_hsl_to_rgb`, `__tmux_lives_cap_valid`, `__tmux_lives_cap_list`, `__tmux_lives_cap_picker`, `__tmux_lives_cap_apply_live`, `__tmux_lives_cap_cmd`, or any `__tcz_cap_*` name may remain in `conf.d/` or `functions/` (grep-guard tests enforce it). `__tcz_theme` (UI chrome) and all `__tmux_lives_theme_*` stay.
- **Fragment argv (locked, renumbered):** 1 `cat` · 2 `pkey` · 3 `skey` · 4 `color` · 5 `invert` · 6 `modalkey` · 7 `scratchkey` · 8 `resizekey` · 9 `statusposkey` · 10 `statusviskey` · 11 `cursorstyle` · **12 `themekey`** · **13 `theme`** · **14 `themephase`** · **15 `themeviv`** · **16 `themeshape`** · **17 `themeease`** · **18 `themerange`**.
- **Picker geometry (locked):** `display-popup -B -E -w 52 -h 20`, frame inner width `IW=50`, frame EXACTLY 20 rows (top · preview · info · sep · 11 rows [10 schemes + off] · sep · 2 key rows · status · bottom). Final row printed WITHOUT a trailing `\n` (the v2 top-border-scroll lesson); guard `count -gt 1` before `printf '%s\e[K\n' $lines[1..-2]`.
- **Phase nudge (locked):** 5° per `←`/`→` press; buffered presses drained into ONE net delta, then ONE single-scheme recompute (net-delta coalescing — NOT the reverted whole-input coalescing).
- **Scheme token list has ONE home:** new `__tmux_lives_theme_schemes` (install side); `theme_valid`, `theme_list`, and the picker batch all consume it.
- **fish gotchas (law, as in the Phase-1 plan):** no comparisons inside `math` (float `test`); multi-value returns `printf '%s\n'` one per line; capture command substitutions into vars before concatenation (`set -a f "…"`, `echo "…"`); `test -n (cmd)` with no output is TRUE — capture+quote; single-quote standalone `#hex` in fragment lines; `%` is integer-only except fish `math`'s signed modulo (fine for phase); pad via a QUOTED var, never inline `(string repeat -n 0 …)`; `set -e` on a possibly-unset var is harmless but guard with `set -q` where a count matters.
- **Test isolation:** live tmux via the `tmux_lives_tmux_socket` seam or the categorize suite's PATH shim ONLY; save/clear ALL touched universals at a section's TOP, restore at the BOTTOM; stub `__tmux_lives_write_fragment` around CLI calls; interactive raw-tty loops are live-smoke only (source-grep + pure-builder tests instead).
- **Suite:** `for t in tests/test-*.fish; fish $t; end` (run via `fish -c` if the invoking shell isn't fish) — all 8 suites ALL PASS before every commit. Test counts DROP in the deletion tasks; that's expected — what matters is ALL PASS and the grep-guards.
- **Commits:** `feat(theme):`/`refactor(theme):` prefixes as given; the user deploys via `fisher update` only.
- Line numbers cited are as of branch `feat/theme-v3-phase2-3` @ `2b09971`; anchor by quoted code, not numbers.
